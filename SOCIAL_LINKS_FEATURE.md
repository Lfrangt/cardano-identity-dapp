# 社交链接功能说明

## 概述

社交链接功能允许用户在去中心化身份档案中添加和展示他们在各个社交平台的账号，帮助建立跨平台的统一身份。

## ✨ 支持的平台

### 国际平台

1. **X (Twitter)** - 𝕏
   - 平台：https://x.com/
   - 格式：@username
   - 示例：@cardano_ada

2. **Instagram** - 📷
   - 平台：https://instagram.com/
   - 格式：@username
   - 示例：@cardano

3. **LinkedIn** - 💼
   - 平台：https://linkedin.com/in/
   - 格式：username
   - 示例：john-doe

4. **GitHub** - 💻
   - 平台：https://github.com/
   - 格式：username
   - 示例：cardano-foundation

5. **Discord** - 🎮
   - 格式：username#1234
   - 示例：user#1234

6. **Telegram** - ✈️
   - 平台：https://t.me/
   - 格式：@username
   - 示例：@cardano

### 中国平台

1. **微信 (WeChat)** - 💬

## 🚀 使用方法

### 添加社交链接

1. **进入身份管理**
   - 连接钱包
   - 点击"👤 身份管理"标签
   - 确保已创建身份档案

2. **打开社交链接管理**
   - 点击"🔗 社交链接"标签
   - 查看当前已添加的链接

3. **添加新链接**
   - 点击"+ 添加链接"按钮
   - 选择社交平台
   - 输入用户名或账号
   - 点击"添加"

4. **自动链接生成**
   - 系统自动根据平台规则生成完整URL
   - 有些平台（如微信）仅显示账号信息

### 编辑和删除

- **删除链接**：鼠标悬停在链接卡片上，点击"✕"按钮
- **重新添加**：删除后可以重新添加相同平台

### 查看社交链接

**在个人档案中：**
- 档案卡片底部显示社交链接图标
- 点击图标可访问对应平台

**在公开档案中：**
- 其他用户查看你的档案时可见
- 可以直接点击访问你的社交账号

## 📊 数据结构

### SocialLink 类型

```typescript
interface SocialLink {
  id: string                    // 唯一标识
  platform: string               // 平台类型
  username: string               // 用户名/账号
  url: string                    // 完整URL或账号信息
  verified: boolean              // 验证状态（规划中）
}
```

### 平台配置

```typescript
{
  id: 'x',                       // 平台ID
  name: 'X (Twitter)',           // 平台全名
  displayName: 'X',              // 显示名称
  icon: '𝕏',                     // 图标
  baseUrl: 'https://x.com/',     // 基础URL
  placeholder: '@username',      // 输入提示
  color: 'from-black to-gray-800' // 渐变颜色
}
```

## 💡 使用技巧

### 1. 选择合适的平台

- 添加你最活跃的社交平台
- 优先添加专业相关的平台
- 保持信息的及时更新

### 2. 统一用户名

- 尽量在不同平台使用相同用户名
- 便于他人识别和记忆
- 建立统一的个人品牌

### 3. 信息真实性

- 确保添加的账号确实属于你
- 避免添加虚假信息
- 保持档案的可信度

### 4. 隐私保护

- 谨慎添加过于私密的联系方式
- 考虑设置档案隐私级别
- 定期审查社交链接

## 🔐 隐私控制

### 可见性规则

社交链接遵循档案的隐私设置：

**公开档案 (public)**
- 所有人可见社交链接
- 可以直接点击访问

**私密档案 (private)**
- 仅自己可见
- 其他人无法看到

**选择性分享 (selective)**
- 基本档案信息公开
- 社交链接对已连接好友可见

### 验证状态（规划中）

未来将支持社交账号验证：
- 通过在社交平台发布特定内容验证
- 验证后显示"✓"标识
- 提高档案可信度

### URL 生成逻辑

```typescript
// 自动生成完整URL
function generateUrl(platform, username) {
  if (username.startsWith('http')) {
    return username  // 已经是完整URL
  }
  
  const cleanUsername = username.replace(/^@/, '')  // 移除 @
  
  if (platform.baseUrl) {
    return platform.baseUrl + cleanUsername
  }
  
  return username  // 无baseUrl的平台直接返回用户名
}
```

## 🔧 故障排除

### 链接无法点击

**原因：** 平台没有配置 baseUrl

**解决：** 
- 检查是否是微信、Discord 等特殊平台
- 这些平台仅显示账号，不生成链接

### 图标显示异常

**原因：** 浏览器不支持某些 Emoji

**解决：**
- 更新浏览器到最新版本
- 使用支持 Emoji 的现代浏览器

### 无法添加链接

**原因：** 用户名格式不正确

**解决：**
- 参考平台的 placeholder 提示
- 移除特殊字符
- 确保用户名有效
